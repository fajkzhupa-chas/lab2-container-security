# Lab 2 - Container Security

## Reflektion
Under denna labb har jag lärt mig att containersäkerhet handlar om att aktivt minimera attackytan, exempelvis genom att använda små bas-images (som slim-versioner) och att aldrig tillåta applikationer att köras som root-användare. Det är också avgörande att definiera exakta versioner av de paket man installerar för att undvika att osäker kod dras ner i framtiden. En SBOM (Software Bill of Materials) är otroligt viktig eftersom den fungerar som en detaljerad innehållsförteckning för containern. Om en ny kritisk sårbarhet upptäcks framöver kan man snabbt söka i sin SBOM för att direkt se om den farliga komponenten finns i miljön, istället för att behöva skanna om allt från noll. Slutligen har jag insett hur policy-enforcement med OPA Gatekeeper fundamentalt förändrar hur man jobbar med Kubernetes. Istället för att bara förlita sig på att utvecklare följer uppsatta säkerhetsriktlinjer, fungerar Gatekeeper som en automatisk dörrvakt. Den blockerar fysiskt alla resurser som bryter mot företagets regler (exempelvis avsaknad av viktiga labels eller försök att köra osäkra containers) innan de ens tillåts starta i klustret.
